# タグ体系

この辞典のタグは、名前からではなく「見た目から探す」ための索引です。タグは増やして構いませんが、似た意味のタグを乱立させると逆引きしにくくなるので、まず既存タグに寄せます。

## 基本カテゴリ

| カテゴリ | 役割 | 例 |
| --- | --- | --- |
| `visual_elements` | 画面に直接見える形や要素 | ネオン、歯車、廃墟、巨大建築、義体 |
| `materials` | 触感や表面素材 | 真鍮、革、コンクリート、ガラス、肉体 |
| `lighting_air` | 光、天候、湿度、空気感 | 蛍光灯、霧、自然光、蝋燭光、宇宙光 |
| `mood` | 感情や雰囲気 | 退廃的、牧歌的、不穏、荘厳、孤独 |
| `settings` | 舞台になる場所 | 宇宙、地下、研究施設、学校、病院、砂漠 |
| `era_technology` | 時代感や技術段階 | 中世、ヴィクトリア朝、戦間期、近未来、崩壊後 |
| `colors` | 色の傾向 | 黒、金、シアン、マゼンタ、アースカラー |
| `motifs` | 検索に使える象徴物 | 探偵、飛行船、太陽光、古代神、サバイバル |

## タグ追加ルール

- 表記は日本語で統一します。
- 似た言葉は、まず既存タグへ寄せます。例: `ネオンサイン` は `ネオン` に寄せる。
- 色は細かくしすぎず、必要になったときだけ増やします。
- `mood` は主観的ですが、逆引きで強いので積極的に使います。
- 特定作品名や固有名詞はタグにしません。必要なら `ai_keywords` やメモに入れます。
- 画像を追加した後も、画像の内容を直接表すタグを増やしすぎず、辞書項目の判定に役立つタグを優先します。

## 逆引きの重み付け案

検索UIを作る場合は、以下の順に重みを付けると使いやすくなります。

| 情報 | 重み | 理由 |
| --- | --- | --- |
| `reverse_lookup.strong_cues` | 高 | その用語を決定づける手がかり |
| `tags.visual_elements` | 高 | 見た目から探すときに一番使う |
| `tags.materials`, `tags.lighting_air` | 中 | 素材と光は雰囲気の判定に強い |
| `tags.mood`, `tags.settings` | 中 | 曖昧検索で候補を広げる |
| `ai_keywords` | 中 | 自由語検索や画像生成の補助 |
| 説明文全文 | 低 | 補助的な全文検索 |

## タグから探す例

| 入力タグ | 優先候補 | 判定理由 |
| --- | --- | --- |
| ネオン、雨、義体、高層ビル | サイバーパンク | 近未来都市と身体拡張が強い |
| ネオン、雨、探偵、路地裏 | ネオンノワール | 犯罪劇と陰影が中心 |
| 歯車、真鍮、蒸気 | スチームパンク | 蒸気機関とヴィクトリア朝風の発明感 |
| 黒煙、装甲、戦間期風 | ディーゼルパンク | 内燃機関と軍事工業感 |
| 蛍光灯、学校、誰もいない | リミナルスペース | 日常空間の不在と違和感 |
| コンクリート、巨大建築、無機質 | ブルータリズム | 打ち放しコンクリートと量塊感 |

## タグの管理方法

`data/visual-dictionary.json` の `tag_catalog` を辞書全体のタグ一覧として使います。新しいタグを使う場合は、先に `tag_catalog` へ追加し、その後で項目側の `tags` に使います。

将来的に項目数が増えたら、`tag_catalog` を `data/tag-taxonomy.json` に分離しても構いません。その場合、辞書データ側はタグ名だけを参照する形にします。
